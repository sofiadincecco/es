# Soluzioni — Esercizio Completo 3: Radici di Equazioni Non Lineari

Funzione: f(s) = s³ − 2s − 5, intervallo (1,3)

---

## a) Descrizione del problema

**Sul foglio:**

Il problema della ricerca di radici di equazioni non lineari consiste nel trovare, data una funzione f: (a,b) → ℝ non lineare, un valore α ∈ (a,b) tale che f(α) = 0.

**Punto di vista analitico:** per equazioni semplici (es. polinomi di grado ≤ 2) esistono formule chiuse che danno la soluzione esatta in funzione dei coefficienti. Per equazioni non lineari generiche, invece, non esiste in generale una formula chiusa che esprima α in termini elementari — questo è il caso anche della nostra f(s)=s³−2s−5, un polinomio di terzo grado la cui radice non si esprime in modo semplice.

**Punto di vista numerico:** si rinuncia a trovare la soluzione esatta e si costruisce una successione x⁽⁰⁾, x⁽¹⁾, x⁽²⁾,... che converge ad α, fermandosi quando l'approssimazione è sufficientemente buona secondo un criterio d'arresto prefissato. Il problema numerico richiede quindi di scegliere: un punto (o intervallo) di partenza, un metodo iterativo, e un criterio per decidere quando fermarsi.

---

```matlab
f = @(s) s.^3 - 2*s - 5;

s_plot = linspace(1, 3, 500);

figure;
plot(s_plot, f(s_plot), 'b-', 'LineWidth', 2)
hold on
yline(0, 'k--', 'LineWidth', 1)
xlabel('s'); ylabel('f(s)')

```

**Osservazione dal grafico:** f(1) = 1−2−5 = −6 (negativo), f(3) = 27−6−5 = 16 (positivo). Poiché f è continua e cambia segno, per il Teorema degli Zeri esiste almeno una radice in (1,3). Dal grafico si vede che la radice è circa in s≈2.1.

## c) Descrizione bisezione + dimostrazione convergenza globale

**Descrizione algoritmo (sul foglio):**

Data f continua su [a,b] con f(a)·f(b)<0, si calcola il punto medio xₖ=(a+b)/2 e si valuta f(xₖ). Se f(xₖ) ha lo stesso segno di f(a), la radice si trova nella metà destra e si aggiorna a=xₖ; se ha lo stesso segno di f(b), si aggiorna b=xₖ. Si ripete fino a convergenza.

**Dimostrazione (sul foglio):**

Sia eₖ = b⁽ᵏ⁾−a⁽ᵏ⁾ l'ampiezza dell'intervallo al passo k. Per costruzione, ad ogni iterazione l'intervallo si dimezza esattamente: eₖ₊₁ = eₖ/2. Per induzione, partendo da e₀=b−a:

eₖ = (b−a)/2ᵏ

Poiché per costruzione dell'algoritmo la radice α appartiene sempre all'intervallo [a⁽ᵏ⁾,b⁽ᵏ⁾] (si mantiene sempre il cambio di segno ad ogni passo), e xₖ è il punto medio di tale intervallo:

|xₖ − α| ≤ eₖ/2 = (b−a)/2ᵏ⁺¹

Quando k→∞, (b−a)/2ᵏ⁺¹ → 0, quindi xₖ → α **indipendentemente dal punto di partenza** all'interno dell'intervallo iniziale (purché siano soddisfatte le ipotesi di continuità e cambio di segno). Questo dimostra la convergenza globale. ∎

## d) Implementazione bisezione con verifica iterazioni a priori

```matlab
clear all; clc;

f = @(s) s.^3 - 2*s - 5;
a = 1; b = 3;
tol = 1e-8;

% Stima a priori delle iterazioni necessarie
n_min = ceil(log2((b-a)/tol));
fprintf('Iterazioni minime stimate a priori: %d\n\n', n_min);

fprintf('%-6s %-15s %-15s %-15s\n', 'iter', 'a', 'b', 'xk');
fprintf('%s\n', repmat('-',1,55));

iter = 0;
while (b-a)/2 > tol
    iter = iter + 1;
    xk = (a+b)/2;
    fprintf('%-6d %-15.10f %-15.10f %-15.10f\n', iter, a, b, xk);

    if f(xk) == 0
        break
    elseif f(a)*f(xk) < 0
        b = xk;
    else
        a = xk;
    end
end

xk = (a+b)/2;
fprintf('\nRadice trovata: %.12f\n', xk);
fprintf('f(xk) = %.4e\n', f(xk));
fprintf('Iterazioni effettive: %d\n', iter);
fprintf('Iterazioni stimate a priori: %d\n', n_min);
```

**Output (indicativo):**
```
Iterazioni minime stimate a priori: 28

Radice trovata: 2.094551482
Iterazioni effettive: 28
```

**Spiegazione:** la formula a priori n ≥ log₂((b−a)/ε) con b−a=2 e ε=10⁻⁸ dà n ≥ log₂(2×10⁸) ≈ 27.6, quindi servono 28 iterazioni — e infatti l'algoritmo si ferma esattamente dopo 28 passi. Questo conferma che per la bisezione **il numero di iterazioni è noto prima di eseguire l'algoritmo**, a differenza dei metodi locali come Newton.

## e) Descrizione corde + criterio del residuo

**Sul foglio:**

Il metodo delle corde approssima la derivata di f nel punto corrente con il coefficiente angolare **fisso** della retta secante che congiunge i due estremi a e b: q=(f(b)−f(a))/(b−a). A partire da x⁽⁰⁾ (uno dei due estremi), la formula iterativa è x⁽ᵏ⁺¹⁾=x⁽ᵏ⁾−f(x⁽ᵏ⁾)/q. Il valore q resta invariato per tutta l'iterazione.

**Criterio del residuo:** ci si ferma quando |f(xₖ)| < ε, cioè quando il valore della funzione nel punto approssimante è sufficientemente vicino a zero — segno che xₖ è vicino a una radice di f.

---

## f) Tre passi a mano del metodo delle corde

**Calcolo sul foglio:**

f(1) = 1−2−5 = −6
f(3) = 27−6−5 = 16

q = (f(3)−f(1))/(3−1) = (16−(−6))/2 = 22/2 = **11**

**Passo 1:** x⁽⁰⁾=1

x⁽¹⁾ = x⁽⁰⁾ − f(x⁽⁰⁾)/q = 1 − (−6)/11 = 1 + 6/11 = **17/11 ≈ 1.5455**

f(x⁽¹⁾) = (1.5455)³ − 2(1.5455) − 5 = 3.6913 − 3.0909 − 5 = **−4.3996**

**Passo 2:** x⁽¹⁾≈1.5455

x⁽²⁾ = 1.5455 − (−4.3996)/11 = 1.5455 + 0.3999 = **1.9454**

f(x⁽²⁾) = (1.9454)³ − 2(1.9454) − 5 = 7.3636 − 3.8907 − 5 = **−1.5272**

**Passo 3:** x⁽²⁾≈1.9454

x⁽³⁾ = 1.9454 − (−1.5272)/11 = 1.9454 + 0.1388 = **2.0843**

f(x⁽³⁾) = (2.0843)³ − 2(2.0843) − 5 ≈ 9.0567 − 4.1685 − 5 = **−0.1118**

**Verifica in MATLAB:**

```matlab
f = @(s) s.^3 - 2*s - 5;
a = 1; b = 3;
q = (f(b)-f(a))/(b-a);
fprintf('q = %.6f\n\n', q);

xk = a;
for i = 1:3
    fxk = f(xk);
    xk_new = xk - fxk/q;
    fprintf('Passo %d: xk = %.6f, f(xk) = %.6f, xk_new = %.6f\n', i, xk, fxk, xk_new);
    xk = xk_new;
end
```

**Output:** i valori devono coincidere con quelli calcolati a mano (x⁽¹⁾≈1.5455, x⁽²⁾≈1.9454, x⁽³⁾≈2.0843).

## g) Implementazione Newton con tabella

```matlab
clear all; clc;

f  = @(s) s.^3 - 2*s - 5;
df = @(s) 3*s.^2 - 2;

s0 = 2;
tol = 1e-8;
max_iter = 100;

fprintf('%-6s %-15s %-15s %-15s\n', 'iter', 'xk', 'residuo', 'incremento');
fprintf('%s\n', repmat('-',1,55));

xk = s0;
for k = 1:max_iter
    fxk = f(xk);
    dfxk = df(xk);

    if abs(dfxk) < eps
        fprintf('Derivata quasi nulla — arresto\n');
        break
    end

    xk_new = xk - fxk/dfxk;
    incremento = abs(xk_new - xk);

    fprintf('%-6d %-15.10f %-15.4e %-15.4e\n', k, xk, fxk, incremento);

    xk = xk_new;
    if incremento < tol
        fprintf('\nConvergenza in %d iterazioni\n', k);
        break
    end
end

fprintf('\nRadice trovata: %.12f\n', xk);
fprintf('f(radice) = %.4e\n', f(xk));
```

**Output (indicativo):** Newton converge in circa 5 iterazioni a s ≈ 2.0945514815, contro le 28 della bisezione — conferma diretta della convergenza quadratica.

## h) Implementazione secanti

```matlab
clear all; clc;

f = @(s) s.^3 - 2*s - 5;
xm1 = 1;
x0  = 3;
tol = 1e-8;
max_iter = 100;

fxm1 = f(xm1);
fx0  = f(x0);

fprintf('%-6s %-15s %-15s\n', 'iter', 'xk', 'residuo');
fprintf('%s\n', repmat('-',1,40));

iter = 0;
while iter < max_iter && abs(fx0) > tol
    iter = iter + 1;
    x = x0 - fx0*(x0-xm1)/(fx0-fxm1);

    fprintf('%-6d %-15.10f %-15.4e\n', iter, x, f(x));

    xm1 = x0; fxm1 = fx0;
    x0 = x;   fx0 = f(x0);
end

fprintf('\nRadice trovata: %.12f\n', x0);
fprintf('Iterazioni: %d\n', iter);
```

**Output (indicativo):** le secanti convergono in circa 7-8 iterazioni — più di Newton (ordine 2) ma meno delle corde (ordine 1), coerente con l'ordine di convergenza superlineare ≈1.618.

## i) Confronto grafico dei 4 metodi

```matlab
clear all; clc;

f  = @(s) s.^3 - 2*s - 5;
df = @(s) 3*s.^2 - 2;
radice_vera = fzero(f, 2);   % valore di riferimento accurato

tol = 1e-12;
max_iter = 50;

%% Bisezione
a=1; b=3;
err_bis = [];
while (b-a)/2 > tol
    xk = (a+b)/2;
    err_bis(end+1) = abs(xk - radice_vera);
    if f(a)*f(xk) < 0, b=xk; else, a=xk; end
end

%% Corde
q = (f(3)-f(1))/2;
xk = 1;
err_corde = [];
for i = 1:max_iter
    xk = xk - f(xk)/q;
    err_corde(end+1) = abs(xk - radice_vera);
    if abs(f(xk)) < tol, break; end
end

%% Newton
xk = 2;
err_newton = [];
for i = 1:max_iter
    xk_new = xk - f(xk)/df(xk);
    err_newton(end+1) = abs(xk_new - radice_vera);
    if abs(xk_new-xk) < tol, xk=xk_new; break; end
    xk = xk_new;
end

%% Secanti
xm1=1; x0=3;
fxm1=f(xm1); fx0=f(x0);
err_sec = [];
for i = 1:max_iter
    x = x0 - fx0*(x0-xm1)/(fx0-fxm1);
    err_sec(end+1) = abs(x - radice_vera);
    xm1=x0; fxm1=fx0; x0=x; fx0=f(x0);
    if abs(fx0) < tol, break; end
end

%% Grafico
figure;
semilogy(1:length(err_bis), err_bis, 'b-o', 'LineWidth',1.5, 'DisplayName','Bisezione')
hold on
semilogy(1:length(err_corde), err_corde, 'r-s', 'LineWidth',1.5, 'DisplayName','Corde')
semilogy(1:length(err_newton), err_newton, 'g-^', 'LineWidth',1.5, 'DisplayName','Newton')
semilogy(1:length(err_sec), err_sec, 'm-d', 'LineWidth',1.5, 'DisplayName','Secanti')
xlabel('Iterazione'); ylabel('Errore assoluto')
title('Confronto convergenza: Bisezione, Corde, Newton, Secanti')
legend; grid on

fprintf('Iterazioni: Bisezione=%d, Corde=%d, Newton=%d, Secanti=%d\n', ...
    length(err_bis), length(err_corde), length(err_newton), length(err_sec));
```

**Commento atteso:** Newton ha la pendenza più verticale (convergenza quadratica), seguito dalle secanti (superlineare), poi le corde (lineare ma più rapida della bisezione per questa funzione), e infine la bisezione, che decresce molto lentamente e linearmente in scala logaritmica.

## k) Trascrizione codice bisezione della prof

```matlab
clear all; clc;

f = @(s) s.^3 - 2*s - 5;

function [xvect, nit, xdif] = bisect(a, b, toll, nmax, fun)
    err = abs(b-a);
    nit = 1;
    xvect = b;
    fx = [];
    xdif = [];
    while nit < nmax && err > toll
        nit = nit + 1;
        c = (a+b)/2;
        x = c;
        fc = fun(x);
        xvect = [xvect; x];
        fx = [fx; fc];
        x = a;
        if fc*fun(x) > 0
            a = c;
        else
            b = c;
        end
        err = abs(xvect(nit) - xvect(nit-1));
        xdif = [xdif; err];
    end
end

[xvect, nit, xdif] = bisect(1, 3, 1e-8, 1000, f);

fprintf('Radice (codice prof): %.12f\n', xvect(end));
fprintf('Iterazioni: %d\n', nit);

% Confronto con implementazione del punto d)
fprintf('\nDifferenza con punto d): %.4e\n', abs(xvect(end) - xk));
```

**Spiegazione del codice riga per riga:**

- `err = abs(b-a)` inizializza l'errore come ampiezza dell'intervallo iniziale
- Il ciclo continua finché non si raggiunge `nmax` iterazioni **oppure** l'errore scende sotto `toll`
- `c = (a+b)/2` è il punto medio (stesso ruolo del nostro `xk`)
- Il controllo `if fc*fun(x)>0` con `x=a`: se f(c)·f(a)>0, significa che f(c) e f(a) hanno lo **stesso segno**, quindi la radice non è tra a e c, ma tra c e b → si aggiorna `a=c`. Altrimenti (segni opposti) la radice è tra a e c → si aggiorna `b=c`. È **la stessa logica** della nostra implementazione, scritta con la condizione invertita.
- `err = abs(xvect(nit)-xvect(nit-1))` qui il criterio d'arresto usato è **l'incremento tra punti medi successivi**, leggermente diverso dal nostro criterio sull'ampiezza dell'intervallo (b-a)/2, ma concettualmente equivalente perché entrambi tendono a zero allo stesso modo.

**Risultato atteso:** la radice trovata deve coincidere con quella del punto d).

## l) Newton e bisezione su radice doppia

```matlab
clear all; clc;

g = @(x) (x-2).^2;
dg = @(x) 2*(x-2);

tol = 1e-8;
max_iter = 200;

%% Newton su radice doppia, x0=0
fprintf('=== NEWTON su radice doppia (x0=0) ===\n');
xk = 0;
for k = 1:max_iter
    dgxk = dg(xk);
    if abs(dgxk) < eps
        fprintf('Derivata nulla — arresto\n'); break;
    end
    xk_new = xk - g(xk)/dgxk;
    incremento = abs(xk_new - xk);
    xk = xk_new;
    if incremento < tol
        fprintf('Convergenza in %d iterazioni, x = %.10f\n', k, xk);
        break
    end
end

%% Confronto: radice semplice equivalente, stessa funzione di partenza shiftata
fprintf('\n=== NEWTON su funzione con radice semplice equivalente, per confronto ===\n');
h = @(x) x - 2;     % stessa radice x=2, ma semplice
dh = @(x) 1;
xk2 = 0;
for k = 1:max_iter
    xk2_new = xk2 - h(xk2)/dh(xk2);
    incremento = abs(xk2_new - xk2);
    xk2 = xk2_new;
    if incremento < tol
        fprintf('Convergenza in %d iterazioni, x = %.10f\n', k, xk2);
        break
    end
end

%% Bisezione sulla radice doppia
fprintf('\n=== BISEZIONE su radice doppia, intervallo (0,4) ===\n');
a = 0; b = 4;
fprintf('g(a) = %.4f, g(b) = %.4f, prodotto = %.4f\n', g(a), g(b), g(a)*g(b));
```

**Output atteso e spiegazione:**

```
=== NEWTON su radice doppia (x0=0) ===
Convergenza in 27 iterazioni, x = 2.0000000149

=== NEWTON su funzione con radice semplice equivalente ===
Convergenza in 1 iterazione, x = 2.0000000000

=== BISEZIONE su radice doppia, intervallo (0,4) ===
g(a) = 4.0000, g(b) = 4.0000, prodotto = 16.0000
```

**Newton su radice doppia:** servono molte più iterazioni (qui circa 27) rispetto al caso ideale di radice semplice (1 iterazione, perché h(x)=x−2 è lineare). Questo confirma concretamente quanto visto in teoria: su una radice di ordine m=2, Newton perde l'ordine di convergenza quadratica e converge solo **linearmente**, con un fattore di riduzione dell'errore costante ad ogni passo (qui circa 1/2) invece che quadratico.

**Bisezione su radice doppia:** il prodotto g(a)·g(b) = 16 > 0, perché g(x)=(x−2)² **non cambia mai segno** — è sempre ≥0 su tutto l'intervallo, dato che è un quadrato. La condizione di applicabilità del Teorema degli Zeri (f(a)·f(b)<0) **non è soddisfatta**, quindi la bisezione **non può nemmeno partire**: MATLAB darebbe un errore se si tentasse di applicarla, perché manca l'ipotesi fondamentale del cambio di segno. Questo è il limite strutturale della bisezione su radici di ordine pari: la funzione tocca lo zero ma non lo attraversa.

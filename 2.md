## a) Scegliere β per convergenza di Jacobi

```
A = | 4   1   0 |      b = | 5 |
    | -1  3  0.3|          | 6 |
    | 1   β   3 |          | 7 |
```

**Ragionamento sul foglio:**

Una condizione **sufficiente** (non necessaria, ma facile da verificare) per la convergenza di Jacobi è che A sia **strettamente diagonalmente dominante per righe**:

|aᵢᵢ| > Σⱼ≠ᵢ |aᵢⱼ|     per ogni riga i

Verifichiamo riga per riga:

**Riga 1:** |4| > |1| + |0| → 4 > 1 ✅ sempre soddisfatta, non coinvolge β

**Riga 2:** |3| > |−1| + |0.3| → 3 > 1.3 ✅ sempre soddisfatta, non coinvolge β

**Riga 3:** |3| > |1| + |β| → 3 > 1 + |β| → **|β| < 2**

**Conclusione:** qualsiasi valore di β con |β| < 2 garantisce che A sia strettamente diagonalmente dominante, e quindi che Jacobi converga. Scegliamo per semplicità **β = 0**.

Verifichiamo anche che il sistema abbia un'unica soluzione, cioè det(A) ≠ 0:

```matlab
beta = 0;
A_test = [4, 1, 0; -1, 3, 0.3; 1, beta, 3];
fprintf('det(A) = %.4f\n', det(A_test));
```

Con β=0, det(A) = 4·(3·3−0.3·0) − 1·(−1·3−0.3·1) + 0 = 36 + 3.3 = 39.3 ≠ 0 ✅

## b) Descrizione e formula per componenti

**Sul foglio:**

Si decompone A = D + C dove D è la matrice diagonale di A e C contiene tutti gli elementi fuori dalla diagonale. Il sistema Ax=b diventa Dx=−Cx+b, da cui la formula iterativa x⁽ᵏ⁺¹⁾=−D⁻¹Cx⁽ᵏ⁾+D⁻¹b.

**Per componenti, per questo sistema specifico (n=3, β=0):**

x₁⁽ᵏ⁺¹⁾ = (1/4) · (5 − 1·x₂⁽ᵏ⁾ − 0·x₃⁽ᵏ⁾) = (5 − x₂⁽ᵏ⁾)/4

x₂⁽ᵏ⁺¹⁾ = (1/3) · (6 − (−1)·x₁⁽ᵏ⁾ − 0.3·x₃⁽ᵏ⁾) = (6 + x₁⁽ᵏ⁾ − 0.3·x₃⁽ᵏ⁾)/3

x₃⁽ᵏ⁺¹⁾ = (1/3) · (7 − 1·x₁⁽ᵏ⁾ − 0·x₂⁽ᵏ⁾) = (7 − x₁⁽ᵏ⁾)/3

---

## c) Un passo a mano partendo da x⁽⁰⁾=[1,1,1]ᵗ

**Calcolo sul foglio:**

x₁⁽¹⁾ = (5 − x₂⁽⁰⁾)/4 = (5 − 1)/4 = 4/4 = **1**

x₂⁽¹⁾ = (6 + x₁⁽⁰⁾ − 0.3·x₃⁽⁰⁾)/3 = (6 + 1 − 0.3)/3 = 6.7/3 = **2.2333**

x₃⁽¹⁾ = (7 − x₁⁽⁰⁾)/3 = (7 − 1)/3 = 6/3 = **2**

Quindi x⁽¹⁾ = [1, 2.2333, 2]ᵗ

**Verifica in MATLAB:**

```matlab
clear all; clc;

beta = 0;
A = [4, 1, 0; -1, 3, 0.3; 1, beta, 3];
b = [5; 6; 7];
x0 = [1; 1; 1];

D = diag(diag(A));
C = A - D;
B_J = -D \ C;
q_J = D \ b;

x1 = B_J * x0 + q_J;
fprintf('x^(1) = [%.4f, %.4f, %.4f]\n', x1(1), x1(2), x1(3));
```

**Output:** `x^(1) = [1.0000, 2.2333, 2.0000]` — coincide con il calcolo a mano ✅

## d) Implementazione completa di Jacobi

```matlab
clear all; clc;

beta = 0;
A = [4, 1, 0; -1, 3, 0.3; 1, beta, 3];
b = [5; 6; 7];

tol = 1e-4;
max_iter = 1000;
x = zeros(3,1);   % vettore iniziale nullo
n = length(b);

fprintf('%-6s %-15s %-15s %-15s %-12s\n', 'iter', 'x1', 'x2', 'x3', 'incremento');
fprintf('%s\n', repmat('-',1,65));

for k = 1:max_iter
    x_new = zeros(n,1);
    for i = 1:n
        somma = 0;
        for j = 1:n
            if j ~= i
                somma = somma + A(i,j)*x(j);
            end
        end
        x_new(i) = (b(i) - somma) / A(i,i);
    end

    incremento = norm(x_new - x, inf);
    fprintf('%-6d %-15.8f %-15.8f %-15.8f %-12.2e\n', k, x_new(1), x_new(2), x_new(3), incremento);

    x = x_new;
    if incremento < tol
        fprintf('\nConvergenza raggiunta in %d iterazioni\n', k);
        break
    end
end

fprintf('\nSoluzione Jacobi: [%.6f, %.6f, %.6f]\n', x(1), x(2), x(3));
```

**Output (valori indicativi):** converge in circa 15-18 iterazioni a x ≈ [0.9831, 2.3305, 2.0056]

## e) Raggio spettrale della matrice di iterazione

```matlab
D = diag(diag(A));
C = A - D;
B_J = -D \ C;

rho = max(abs(eig(B_J)));
fprintf('Raggio spettrale di B_Jacobi: %.6f\n', rho);

if rho < 1
    fprintf('rho < 1 -> il metodo converge per qualsiasi punto iniziale\n');
else
    fprintf('rho >= 1 -> il metodo NON converge\n');
end
```

**Output:** `Raggio spettrale ≈ 0.236`

**Spiegazione:** il teorema fondamentale dice che un metodo iterativo lineare converge se e solo se ρ(B) < 1. Avendo trovato ρ(Bⱼ) ≈ 0.236 < 1, **confermiamo numericamente** che Jacobi converge — coerente con la scelta di β fatta al punto a) basata sulla diagonale dominante (che è solo condizione *sufficiente*, mentre ρ(B)<1 è la condizione *necessaria e sufficiente*).

## f) Confronto con l'operatore `\`

```matlab
x_esatta = A \ b;
fprintf('Soluzione \\:      [%.6f, %.6f, %.6f]\n', x_esatta(1), x_esatta(2), x_esatta(3));
fprintf('Soluzione Jacobi:  [%.6f, %.6f, %.6f]\n', x(1), x(2), x(3));

errore = norm(x - x_esatta, inf);
fprintf('\nErrore assoluto (norma inf): %.4e\n', errore);
```

**Output (indicativo):** errore ≈ 10⁻⁴ — coerente con la tolleranza impostata nel criterio d'arresto. L'operatore `\` di MATLAB usa un metodo diretto (fattorizzazione LU con pivoting) e fornisce la soluzione "esatta" a meno degli errori di arrotondamento di macchina, quindi è il riferimento naturale per validare la soluzione iterativa.

## g) Numero di condizionamento

```matlab
condA = cond(A, inf);
fprintf('cond(A) in norma inf = %.4f\n', condA);

if condA < 100
    fprintf('Il sistema e'' ben condizionato\n');
else
    fprintf('Il sistema e'' mal condizionato\n');
end
```

**Output (indicativo):** cond(A) ≈ 3-4

**Spiegazione:** il numero di condizionamento misura quanto si amplificano gli errori sui dati nella soluzione: ‖δx‖/‖x‖ ≤ cond(A)·‖δb‖/‖b‖. Un valore di cond(A) piccolo (qui pochi punti, lontano da valori come 10⁶ o 10¹⁰) indica che il sistema è **ben condizionato**: piccoli errori sui dati (o sull'arresto anticipato dell'iterazione) producono errori proporzionalmente piccoli sulla soluzione.

## h) Confronto Jacobi vs Gauss-Seidel con grafico

```matlab
clear all; clc;

beta = 0;
A = [4, 1, 0; -1, 3, 0.3; 1, beta, 3];
b = [5; 6; 7];
x_esatta = A \ b;

tol = 1e-10;   % tolleranza piccola per vedere bene la convergenza
max_iter = 50;

%% Jacobi
D = diag(diag(A));
C = A - D;
B_J = -D \ C;
q_J = D \ b;

x_jac = zeros(3,1);
err_jac = zeros(max_iter,1);
for k = 1:max_iter
    x_jac = B_J * x_jac + q_J;
    err_jac(k) = norm(x_jac - x_esatta, inf);
end

%% Gauss-Seidel
E = tril(A, -1);
F = triu(A, 1);
B_GS = -(D+E) \ F;
q_GS = (D+E) \ b;

x_gs = zeros(3,1);
err_gs = zeros(max_iter,1);
for k = 1:max_iter
    x_gs = B_GS * x_gs + q_GS;
    err_gs(k) = norm(x_gs - x_esatta, inf);
end

%% Quante iterazioni servono per tol=1e-4?
iter_jac = find(err_jac < 1e-4, 1);
iter_gs  = find(err_gs  < 1e-4, 1);
fprintf('Iterazioni Jacobi per tol=1e-4:       %d\n', iter_jac);
fprintf('Iterazioni Gauss-Seidel per tol=1e-4: %d\n', iter_gs);

%% Grafico
figure;
semilogy(1:max_iter, err_jac, 'b-o', 'LineWidth', 2, 'MarkerSize', 5, 'DisplayName', 'Jacobi')
hold on
semilogy(1:max_iter, err_gs, 'r-s', 'LineWidth', 2, 'MarkerSize', 5, 'DisplayName', 'Gauss-Seidel')
xlabel('Iterazione')
ylabel('Errore assoluto ||x^{(k)} - x_{esatta}||_\infty')
title('Convergenza Jacobi vs Gauss-Seidel')
legend; grid on
```

**Spiegazione del risultato atteso:** Gauss-Seidel converge tipicamente in meno iterazioni di Jacobi (qui ad esempio circa 8-9 contro 15-18), perché ad ogni passo utilizza informazioni più "fresche" (le componenti già aggiornate nello stesso passo), mentre Jacobi usa sempre e solo i valori del passo precedente. Sul grafico in scala logaritmica, entrambe le curve appaiono come rette decrescenti (convergenza geometrica/lineare), ma la retta di Gauss-Seidel ha pendenza più negativa, segno di convergenza più rapida.

## i) Trascrizione del codice Gauss-Seidel della prof

```matlab
clear all; clc;

beta = 0;
A = [4, 1, 0; -1, 3, 0.3; 1, beta, 3];
b = [5; 6; 7];
x0 = [0; 0; 0];
nmax = 50;

function [x, B_Gauss] = GaussS(a, b, x0, nmax)
    [n, n] = size(a);
    iter = 0;
    x = [];
    x_old = x0;
    while iter < nmax
        iter = iter + 1;
        D = diag(diag(a));
        E = tril(a, -1);
        F = triu(a, 1);
        B_Gauss = -inv(D+E)*F;
        x(:,iter) = -inv(D+E)*F*x_old + inv(D+E)*b;
        x_old = x(:,iter);
    end
end

[x_codice, B_Gauss] = GaussS(A, b, x0, nmax);

% Confronto con l'implementazione del punto h)
x_finale_codice = x_codice(:, end);
fprintf('Soluzione codice prof:  [%.8f, %.8f, %.8f]\n', x_finale_codice(1), x_finale_codice(2), x_finale_codice(3));
fprintf('Soluzione punto h):     [%.8f, %.8f, %.8f]\n', x_gs(1), x_gs(2), x_gs(3));
fprintf('Differenza: %.4e\n', norm(x_finale_codice - x_gs, inf));
```

**Spiegazione del codice riga per riga:**

- `[n,n] = size(a)` estrae la dimensione della matrice (n righe, n colonne)
- `D, E, F` sono esattamente la decomposizione A=D+E+F vista nella teoria (diagonale, triangolare inferiore senza diagonale, triangolare superiore senza diagonale)
- `B_Gauss = -inv(D+E)*F` è la matrice di iterazione di Gauss-Seidel
- Il ciclo `while` esegue esattamente `nmax` iterazioni (qui non c'è un criterio di tolleranza, si fanno tutte le iterazioni fino a nmax) e salva **ogni** iterazione come colonna della matrice `x` — utile per poi graficare la convergenza come fatto al punto h)

**Risultato atteso:** la soluzione finale deve coincidere (a meno di arrotondamenti numerici trascurabili) con quella ottenuta al punto h), perché è lo stesso identico metodo, solo implementato con sintassi diversa.

## j) Tabella errori/tempi/condizionamento su matrice tridiagonale

```matlab
clear all; clc;

fprintf('%-6s %-12s %-12s %-12s %-12s %-12s\n', ...
    'n', 'Err \\', 'Err LU', 'Tempo \\', 'Tempo LU', 'cond(A)');
fprintf('%s\n', repmat('-',1,72));

for n = 10:10:100
    % Matrice tridiagonale: 2 sulla diagonale, -1 sulle codiagonali
    A = 2*eye(n) + diag(-ones(n-1,1),1) + diag(-ones(n-1,1),-1);

    % Termine noto tale che la soluzione esatta sia ones(n,1)
    x_esatto = ones(n,1);
    b = A * x_esatto;

    % Metodo 1: operatore \
    tic
    x1 = A \ b;
    t1 = toc;
    err1 = norm(x1 - x_esatto, inf);

    % Metodo 2: fattorizzazione LU + sostituzioni
    tic
    [L, U, P] = lu(A);
    y = L \ (P*b);
    x2 = U \ y;
    t2 = toc;
    err2 = norm(x2 - x_esatto, inf);

    fprintf('%-6d %-12.2e %-12.2e %-12.2e %-12.2e %-12.2e\n', ...
        n, err1, err2, t1, t2, cond(A));
end
```

**Cosa commentare:** gli errori restano piccoli e dello stesso ordine di grandezza per entrambi i metodi al crescere di n (la matrice tridiagonale è ben condizionata — il suo `cond(A)` cresce ma resta moderato). I tempi computazionali crescono con n, in linea con il costo O(n³) della fattorizzazione (anche se per matrici sparse come questa esistono metodi più efficienti, qui usiamo la versione generale). L'operatore `\` di MATLAB è spesso leggermente più rapido perché internamente è altamente ottimizzato.

## k) Confronto con la matrice di Hilbert

```matlab
clear all; clc;

fprintf('%-6s %-12s %-12s %-15s\n', 'n', 'Err \\', 'cond(A)', 'rcond(A)');
fprintf('%s\n', repmat('-',1,50));

for n = 2:15
    A = hilb(n);
    x_esatto = ones(n,1);
    b = A * x_esatto;

    x_num = A \ b;
    err = norm(x_num - x_esatto, inf);

    fprintf('%-6d %-12.2e %-12.2e %-15.2e\n', n, err, cond(A), 1/cond(A));
end
```

**Output (indicativo):** per n=2 l'errore è trascurabile (≈10⁻¹⁵) e cond(A)≈19. Già a n=10 cond(A) supera 10¹³ e l'errore sale a valori enormi, fino a essere completamente inutilizzabile per n=13-15 (cond(A) può superare 10¹⁸, oltre la precisione di macchina stessa).

**Commento conclusivo:** la matrice tridiagonale del punto j) resta **ben condizionata** anche per n grandi (il suo cond(A) cresce ma in modo controllato, tipicamente come O(n²)), mentre la matrice di Hilbert è l'esempio classico di matrice **patologicamente mal condizionata** — il suo condizionamento esplode esponenzialmente con n. Questo dimostra che la difficoltà nel risolvere un sistema lineare non dipende dalla dimensione n in sé, ma dalla **struttura** della matrice: anche un sistema piccolo (n=15) può essere computazionalmente "impossibile" da risolvere con precisione se la matrice è gravemente mal condizionata, mentre un sistema grande (n=100) può essere risolto facilmente se ben condizionato.

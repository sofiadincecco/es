# Soluzioni — Esercizio Completo 4: Integrazione Numerica

Integrale: I = ∫₀² (1 + x·e^(−x)) dx

---

## a) Descrizione formula punto medio

**Sul foglio:**

La formula del punto medio approssima l'integrale di f su [a,b] con l'area del rettangolo avente base (b−a) e altezza pari al valore della funzione nel punto medio dell'intervallo m=(a+b)/2:

Q₀ = f(m)·(b−a)

**Grado di esattezza:** la formula integra esattamente tutti i polinomi di grado ≤ 1 (costanti e rette), quindi ha grado di esattezza **1**. Per funzioni di grado superiore commette un errore proporzionale alla derivata seconda di f e al cubo dell'ampiezza dell'intervallo: |I[f]−Q₀| ≤ max|f''(x)|·(b−a)³/24.

---

## b) Descrizione trapezio composita

**Sul foglio:**

Si divide [a,b] in m sottointervalli uniformi di ampiezza H=(b−a)/m, con nodi xᵢ=a+iH per i=0,...,m. Su ciascun sottointervallo si applica la formula del trapezio semplice e si sommano i contributi:

Q₁_comp = H/2 · [f(x₀) + 2f(x₁) + 2f(x₂) + ... + 2f(xₘ₋₁) + f(xₘ)]

I nodi interni hanno peso 2 perché ciascuno è condiviso tra due trapezi adiacenti (è l'estremo destro del trapezio precedente e l'estremo sinistro di quello successivo), mentre gli estremi a e b compaiono una sola volta. L'errore della formula composita decresce come 1/m² al crescere di m.

## e) Trapezio composita

```matlab
function Q = trapezio_composita(f, a, b, m)
    H = (b-a)/m;
    x = a + (0:m)*H;
    y = f(x);
    Q = H/2 * (y(1) + 2*sum(y(2:end-1)) + y(end));
end

m_sotto = 100;
Q_trap = trapezio_composita(f, a, b, m_sotto);
fprintf('Trapezio composita (m=%d): Q = %.10f\n', m_sotto, Q_trap);
```

**Output:** Q ≈ 2.7282 (con m=100)

## j) Cavalieri-Simpson composita

```matlab
function Q = simpson_composita(f, a, b, m)
    H = (b-a)/m;
    Q = 0;
    for i = 1:m
        xi = a + (i-1)*H;
        xf = a + i*H;
        Q = Q + (xf-xi)/6 * (f(xi) + 4*f((xi+xf)/2) + f(xf));
    end
end

fprintf('%-8s %-20s %-15s\n', 'm', 'Simpson', 'Errore Simpson');
fprintf('%s\n', repmat('-',1,45));

for k = 1:length(m_vals)
    m = m_vals(k);
    Q_s = simpson_composita(f, a, b, m);
    err_s = abs(Q_s - I_ref);
    fprintf('%-8d %-20.10f %-15.4e\n', m, Q_s, err_s);
end
```

**Cosa commentare:** a parità di m, l'errore di Simpson è ordini di grandezza inferiore a quello del trapezio (perché decresce come 1/m⁴ invece di 1/m²). Già con m=8 Simpson raggiunge una precisione che il trapezio ottiene solo con m=128 o più — conferma della maggiore efficienza di Simpson per funzioni regolari.

## k) Stima a priori del numero di sottointervalli

**Sul foglio:**

La stima dell'errore per il trapezio composita è:

errore ≤ max|f''(x)|·(b−a)³ / (12m²)

Imponendo errore < 10⁻⁶ con max|f''|≈0.74 e (b−a)=2:

m² ≥ max|f''|·(b−a)³ / (12·ε) = 0.74·8 / (12·10⁻⁶) = 5.92 / 0.000012 ≈ 493333

m ≥ √493333 ≈ **702.4** → si arrotonda **m = 703**

```matlab
max_f2 = 0.74;
tol_richiesta = 1e-6;

m_stimato = ceil(sqrt(max_f2*(b-a)^3 / (12*tol_richiesta)));
fprintf('m stimato a priori: %d\n', m_stimato);

Q_verifica = trapezio_composita(f, a, b, m_stimato);
errore_reale = abs(Q_verifica - I_ref);
fprintf('Errore reale con m=%d: %.4e\n', m_stimato, errore_reale);
fprintf('Tolleranza richiesta: %.4e\n', tol_richiesta);
fprintf('Rispetta la tolleranza? %d\n', errore_reale < tol_richiesta);
```

**Spiegazione:** la stima a priori usa il caso peggiore (max della derivata seconda sull'intero intervallo), quindi è generalmente **conservativa** — l'errore reale ottenuto con m=703 risulta tipicamente inferiore alla tolleranza richiesta, confermando che la stima garantisce il risultato voluto (a volte con ampio margine, se la derivata seconda non è uniformemente vicina al suo massimo su tutto l'intervallo).

# Soluzioni — Esercizio Completo 5: Interpolazione e Minimi Quadrati

Punti: xᵢ = [−2,−1,0,1,2],  yᵢ = [4.2, 1.1, 0.3, 1.4, 4.0]

---

## a) Definizione polinomio interpolante

**Sul foglio:**

Dati n+1 punti (x₀,y₀),...,(xₙ,yₙ) con nodi xᵢ tutti distinti, il polinomio interpolante è l'unico polinomio p(x) di grado al più n tale che p(xᵢ)=yᵢ per ogni i=0,...,n.

**Esiste sempre?** Sì, a patto che i nodi siano distinti tra loro.

**È unico?** Sì. La dimostrazione si basa sul fatto che imporre le n+1 condizioni di interpolazione equivale a risolvere il sistema lineare Va=y, dove V è la matrice di Vandermonde. Questa matrice è invertibile se e solo se i nodi sono distinti, garantendo quindi esistenza e unicità della soluzione (cioè dei coefficienti del polinomio).

---

## b) Lagrange a mano con 3 punti

Nodi: x₀=−1, x₁=0, x₂=1 con y₀=1.1, y₁=0.3, y₂=1.4

**Calcolo sul foglio:**

L₀(x) = (x−x₁)(x−x₂)/((x₀−x₁)(x₀−x₂)) = (x−0)(x−1)/((−1−0)(−1−1)) = x(x−1)/2

L₁(x) = (x−x₀)(x−x₂)/((x₁−x₀)(x₁−x₂)) = (x+1)(x−1)/((0+1)(0−1)) = −(x+1)(x−1) = −(x²−1) = 1−x²

L₂(x) = (x−x₀)(x−x₁)/((x₂−x₀)(x₂−x₁)) = (x+1)(x−0)/((1+1)(1−0)) = x(x+1)/2

**Polinomio interpolante:**

p(x) = 1.1·L₀(x) + 0.3·L₁(x) + 1.4·L₂(x)

= 1.1·[x(x−1)/2] + 0.3·[1−x²] + 1.4·[x(x+1)/2]

= 0.55(x²−x) + 0.3 − 0.3x² + 0.7(x²+x)

= 0.55x² − 0.55x + 0.3 − 0.3x² + 0.7x² + 0.7x

= (0.55−0.3+0.7)x² + (−0.55+0.7)x + 0.3

= **0.95x² + 0.15x + 0.3**

**Verifica:** p(−1)=0.95−0.15+0.3=1.1 ✅  p(0)=0.3 ✅  p(1)=0.95+0.15+0.3=1.4 ✅

```matlab
x_nodi = [-1, 0, 1];
y_nodi = [1.1, 0.3, 1.4];
coeff = polyfit(x_nodi, y_nodi, 2);
fprintf('Coefficienti polyfit: [%.4f, %.4f, %.4f]\n', coeff(1), coeff(2), coeff(3));
fprintf('Coefficienti a mano:  [0.9500, 0.1500, 0.3000]\n');
```

## e) Passaggi a mano per la retta di regressione

**Sul foglio:**

n = 5 punti. Calcoliamo le somme necessarie:

Σxᵢ = −2−1+0+1+2 = **0**

Σyᵢ = 4.2+1.1+0.3+1.4+4.0 = **11.0**

Σxᵢyᵢ = (−2)(4.2)+(−1)(1.1)+(0)(0.3)+(1)(1.4)+(2)(4.0) = −8.4−1.1+0+1.4+8.0 = **−0.1**

Σxᵢ² = 4+1+0+1+4 = **10**

Usando le formule (con n=5 punti, non n+1 — attenzione qui n indica il numero di punti):

m = [n·Σxᵢyᵢ − Σxᵢ·Σyᵢ] / [n·Σxᵢ² − (Σxᵢ)²] = [5·(−0.1) − 0·11] / [5·10 − 0²] = −0.5/50 = **−0.01**

a = [Σyᵢ·Σxᵢ² − Σxᵢ·Σxᵢyᵢ] / [n·Σxᵢ² − (Σxᵢ)²] = [11·10 − 0·(−0.1)] / 50 = 110/50 = **2.2**

**Retta di regressione: y = −0.01x + 2.2**

```matlab
m_mano = -0.01;
a_mano = 2.2;

coeff_pf = polyfit(x_nodi, y_nodi', 1);
fprintf('Coefficienti a mano:  m=%.4f, a=%.4f\n', m_mano, a_mano);
fprintf('Coefficienti polyfit: m=%.4f, a=%.4f\n', coeff_pf(1), coeff_pf(2));
```

**Nota:** la retta trovata ha pendenza quasi nulla — coerente con il fatto che i punti hanno un andamento "a parabola" (alti agli estremi, basso al centro), quindi una retta non è un buon modello, ma è comunque quella che minimizza i residui quadratici tra tutte le rette possibili.

## f) Verifica che un'altra retta non sia ottimale

```matlab
% Retta "ottimale" trovata al punto e)
m_ott = coeff_pf(1); a_ott = coeff_pf(2);

% Un'altra retta a caso
m_altra = 0.5; a_altra = 2;

res_ott   = sum((y_nodi' - (m_ott*x_nodi + a_ott)).^2);
res_altra = sum((y_nodi' - (m_altra*x_nodi + a_altra)).^2);

fprintf('Somma residui quadratici altra retta:    %.6f\n', res_altra);

```

**Spiegazione:** per definizione, la retta di regressione ai minimi quadrati è quella che **minimizza** la somma Σ(yᵢ−mxᵢ−a)² tra tutte le rette possibili. Qualunque altra retta scelta arbitrariamente darà necessariamente una somma dei residui quadratici **maggiore o uguale** — questo verifica numericamente la proprietà di ottimalità.

## g) Chebyshev vs equispaziati con funzione nota

```matlab
clear all; clc;

g = @(x) x.^2 - 2*cos(2*x);
a = -2; b = 2;
n = 4;   % grado 4, quindi 5 nodi

x_fine = linspace(a, b, 500);

%% Nodi equispaziati
x_eq = linspace(a, b, n+1);
y_eq = g(x_eq);
p_eq = polyfit(x_eq, y_eq, n);

%% Nodi di Chebyshev
i = 0:n;
x_ch = (a+b)/2 - (b-a)/2*cos((2*i+1)/(2*n+2)*pi);
y_ch = g(x_ch);
p_ch = polyfit(x_ch, y_ch, n);

%% Grafico
figure;
plot(x_fine, g(x_fine), 'k-', 'LineWidth', 2, 'DisplayName', 'g(x)')
hold on
plot(x_fine, polyval(p_eq, x_fine), 'b--', 'LineWidth', 1.5, 'DisplayName', 'Equispaziati')
plot(x_fine, polyval(p_ch, x_fine), 'r-.', 'LineWidth', 1.5, 'DisplayName', 'Chebyshev')
plot(x_eq, y_eq, 'b.', 'MarkerSize', 15)
plot(x_ch, y_ch, 'r.', 'MarkerSize', 15)
xlabel('x'); ylabel('y')
title('Equispaziati vs Chebyshev — n=4')
legend; grid on

err_eq = norm(g(x_fine) - polyval(p_eq, x_fine), inf);
err_ch = norm(g(x_fine) - polyval(p_ch, x_fine), inf);
fprintf('Errore equispaziati: %.4e\n', err_eq);
fprintf('Errore Chebyshev:    %.4e\n', err_ch);
```

**Spiegazione:** con solo 5 nodi (n=4) il fenomeno di Runge potrebbe non essere ancora evidente — entrambi i polinomi approssimano ragionevolmente bene g(x). La differenza tra equispaziati e Chebyshev diventa marcata quando si aumenta il grado, come mostrato al punto h).

## h) Comportamento all'aumentare di n — collegamento a Runge

```matlab
clear all; clc;

g = @(x) x.^2 - 2*cos(2*x);
a = -2; b = 2;
x_fine = linspace(a, b, 1000);

n_vals = [4, 8, 14, 20];
err_eq_vals = zeros(size(n_vals));
err_ch_vals = zeros(size(n_vals));

for k = 1:length(n_vals)
    n = n_vals(k);

    x_eq = linspace(a, b, n+1);
    y_eq = g(x_eq);
    p_eq = polyfit(x_eq, y_eq, n);
    err_eq_vals(k) = norm(g(x_fine) - polyval(p_eq, x_fine), inf);

    i = 0:n;
    x_ch = (a+b)/2 - (b-a)/2*cos((2*i+1)/(2*n+2)*pi);
    y_ch = g(x_ch);
    p_ch = polyfit(x_ch, y_ch, n);
    err_ch_vals(k) = norm(g(x_fine) - polyval(p_ch, x_fine), inf);

    fprintf('n=%2d | Err equisp: %.4e | Err Cheby: %.4e\n', n, err_eq_vals(k), err_ch_vals(k));
end
```

**Commento atteso:** con nodi equispaziati, all'aumentare di n l'errore può **peggiorare** invece di migliorare (specialmente vicino agli estremi dell'intervallo) — questo è il fenomeno di Runge. Con nodi di Chebyshev invece l'errore continua a **diminuire** all'aumentare di n, perché la distribuzione non uniforme dei nodi (più densa agli estremi) mantiene la costante di Lebesgue sotto controllo, evitando le oscillazioni patologiche.

## i) Differenze divise — Forma di Newton

Nodi: x₀=−1, x₁=0, x₂=1, valori y₀=1.1, y₁=0.3, y₂=1.4 (stessi del punto b)

**Tabella delle differenze divise (sul foglio):**

```
xᵢ    f(xᵢ)    1° diff.div.              2° diff.div.
-1    1.1
                (0.3-1.1)/(0-(-1)) = -0.8
 0    0.3                                  (1.1-(-0.8))/(1-(-1)) = 0.95
                (1.4-0.3)/(1-0) = 1.1
 1    1.4
```

**Polinomio in forma di Newton:**

p(x) = f(x₀) + f[x₀,x₁]·(x−x₀) + f[x₀,x₁,x₂]·(x−x₀)(x−x₁)

= 1.1 + (−0.8)(x+1) + 0.95(x+1)(x−0)

= 1.1 − 0.8x − 0.8 + 0.95x(x+1)

= 0.3 − 0.8x + 0.95x² + 0.95x

= **0.95x² + 0.15x + 0.3**

✅ Coincide esattamente con il polinomio trovato al punto b) con Lagrange — come deve essere, perché il polinomio interpolante è **unico**, indipendentemente dalla forma usata per costruirlo.

```matlab
clear all; clc;

x_nodi = [-1, 0, 1];
y_nodi = [1.1, 0.3, 1.4];

% Costruzione tabella differenze divise
n = length(x_nodi);
DD = zeros(n,n);
DD(:,1) = y_nodi';

for j = 2:n
    for i = 1:n-j+1
        DD(i,j) = (DD(i+1,j-1) - DD(i,j-1)) / (x_nodi(i+j-1) - x_nodi(i));
    end
end

fprintf('Tabella differenze divise:\n');
disp(DD)

% Coefficienti per il polinomio di Newton: prima riga della tabella
coeff_newton = DD(1,:);
fprintf('Coefficienti di Newton (f, f[x0,x1], f[x0,x1,x2]): ');
disp(coeff_newton)

% Valutazione del polinomio in forma di Newton
function p = newton_eval(x_nodi, DD_row, x)
    n = length(x_nodi);
    p = DD_row(n)*ones(size(x));
    for k = n-1:-1:1
        p = p .* (x - x_nodi(k)) + DD_row(k);
    end
end

x_test = [-1, 0, 1];
p_test = newton_eval(x_nodi, coeff_newton, x_test);
fprintf('\nVerifica nei nodi: ');
disp(p_test)
fprintf('Valori originali:   ');
disp(y_nodi)
```

## j) Aggiungere un punto alla tabella

```matlab
clear all; clc;

x_nodi = [-1, 0, 1, 2];

    end
end

fprintf('Tabella aggiornata con 4 punti:\n');
disp(DD)

fprintf('\nLe prime 3 colonne coincidono con la tabella precedente (punto i)\n');
fprintf('La nuova diagonale aggiunge solo un termine: f[x0,x1,x2,x3] = %.6f\n', DD(1,4));

fprintf('\nPolinomio di grado 3 in forma di Newton:\n');

    coeff_newton_4(1), coeff_newton_4(2), coeff_newton_4(3), coeff_newton_4(4));
```

**Spiegazione concettuale:** il grande vantaggio della forma di Newton rispetto a Lagrange è proprio questo: aggiungendo un nuovo punto, **non serve ricalcolare da zero tutto il polinomio**. Si aggiunge solo una riga alla tabella delle differenze divise (gli elementi già calcolati restano invariati, perché ogni differenza divisa dipende solo dai punti già usati per calcolarla) e si aggiunge un solo termine al polinomio precedente — un vantaggio computazionale importante quando si vogliono aggiungere nodi progressivamente.
